Project's Report

   This week we had to add some more functionalities to our first web application
-the currency converter and build it again using servlets.

   Actually we had to add a button that would redirect to a page that provides
the report of our project and one more button that would let download the source code.
The basic functionality of the conversion of the requested amount to another currency
is done in the conv.java file by requesting and getting the live currencies from Google.
After getting the google response I parse it in order to generate a string that is the result
and I redirect it to the convRes.jsp page that shows it to the user.

  The functionality of downloading the source code by clicking the appropriate
button is done in the srcServlet.java file and follows exactly the instructions represented
in Chapter 4 of the book (Head First Servlets and JSP,2nd Edition),page 163.

  The other button- the report button redirects the user to a page that represents
the report(way of implementation) of the project. This was requested to be done using Listener.
It is done in repCtxListener.java repServlet.java myRep.java(attribute class) files
while accessing the web-apps context parameter, where the link to this report.html page is defined.
